# 快餐厨房智能管理系统（Fastfood Kitchen）

## 项目概述

快餐厨房智能管理系统是一套面向**称重式快餐连锁门店**的内部管理平台，服务于厨房团队（厨师长、厨师、切配、早餐师、采购、店长、管理员），覆盖从菜品规划、备料管理、成本核算到经营分析的完整业务链路。

**核心定位：** 通过数据驱动的推荐算法，帮助门店管理者做出更优的菜单决策，提升客单价稳定性与门店利润。

**业务模式：** 荤素同价、按两计价的自选快餐。菜品统一陈列在餐线上，顾客自行夹取后称重结算（如原价 3.38 元/两）。不同门店根据消费能力定价不同。

## 功能模块

### 一、菜品与食材管理

**菜品库（118 道菜品，9 大分类）**


| 分类    | 说明              | 菜品数（示例） |
| ----- | --------------- | ------- |
| 蒸菜    | 粉蒸肉、清蒸鲈鱼、蒸水蛋等   | 17      |
| 炒菜    | 小炒肉、红烧茄子、土豆丝等   | 46      |
| 煎扒/油炸 | 煎鸡扒、炸鸡翅根、炸糍粑等   | 11      |
| 砂锅    | 红烧肉、酸菜鱼、牛腩炖萝卜等  | 12      |
| 凉菜    | 白切鸡、凉拌木耳、擂辣椒皮蛋等 | 13      |
| 例汤    | 玉米骨汤、冬瓜薏米汤等     | 6       |
| 水果    | 西瓜、哈密瓜、葡萄等      | 4       |
| 茶饮    | 三花茶、罗汉果茶、甘蔗茅根茶等 | 6       |
| 早餐    | 皮蛋瘦肉粥、鲜肉包、葱油饼等  | 3       |


每道菜品包含：食材组成（BOM）、烹饪步骤（SOP）、成本核算、分类标签（大荤/小荤/素菜）、推荐权重。

**食材库（273 种食材，17 个分类）**

覆盖蔬菜、肉类、禽类、水产海鲜、菌菇、豆制品、调味配菜、杂粮、水果、茶饮等全品类，每项记录采购价、库存状态、保质属性。

### 二、智能推荐算法

推荐系统为厨师长的菜单规划提供**多维度评分建议**，综合考虑以下因素：

```
菜品推荐分 = 毛利基础分
           + 管理者推荐权重
           + 库存状态加成（优先消耗易腐食材）
           + 客单价反馈加成（基于前一天经营数据动态调整）
           + 菜单搭配缺口加成（补齐大荤/小荤/素菜缺口）
           + 菜品历史表现加成（剩余量反馈）
           + 食材多样性加成（避免属性重复）
           + 分类均衡加成（避免单一类型过多）
           - 近期频率惩罚（避免重复出菜）
```

**客单价反馈机制：**


| 场景      | 系统响应                | 目标          |
| ------- | ------------------- | ----------- |
| 客单价低于目标 | 提升荤菜推荐权重，降低饱腹型素菜权重  | 提升吸引力，拉动拿取量 |
| 客单价高于目标 | 提升高毛利菜推荐权重，降低低毛利菜权重 | 优化利润结构      |
| 正常区间    | 不额外调整               | 维持现状        |


### 三、菜单评分引擎

对每日菜单进行 4 维度综合评分（满分 100）：


| 维度    | 权重   | 评估内容          |
| ----- | ---- | ------------- |
| 菜单完整度 | 30 分 | 各分类菜品数是否达标    |
| 食材多样性 | 30 分 | 食材属性覆盖度，避免重复  |
| 菜品新鲜度 | 20 分 | 近期未出现菜品占比     |
| 毛利结构  | 20 分 | 高/中/引流款比例是否合理 |


### 四、每日经营数据闭环

```
店长填报（前一天数据）        系统分析                次日菜单建议
┌─────────────────┐    ┌──────────────┐    ┌───────────────┐
│ • 早餐/正餐客单价 │───▶│ 客单价偏差分析 │───▶│ 推荐排序调整   │
│ • 就餐人数       │    │ 搭配缺口分析   │    │ 搭配缺口提示   │
│ • 天气情况       │    │ 趋势判断      │    │ 参数调优建议   │
│ • 菜品剩余反馈   │    │ 菜品表现统计   │    │ 异常预警       │
└─────────────────┘    └──────────────┘    └───────────────┘
```

- 数据每日填报，早餐与正餐分开统计
- 菜品剩余反馈仅记录突出问题（剩余量大或销售极差），通报至采购和厨师长
- 支持补填历史数据

### 五、门店配置（门店隔离）

每个门店独立配置：


| 配置项   | 说明                           |
| ----- | ---------------------------- |
| 目标客单价 | 早餐/正餐分别设置                    |
| 称重单价  | 原价与会员价（元/两）                  |
| 米饭价格  | 按位计费，独立于菜品体系                 |
| 菜单标准  | 各分类需要多少道菜（蒸菜 6 道、炒菜 10 道...） |
| 搭配规则  | 大荤/小荤/素菜数量要求                 |
| 固定菜品  | 按星期几配置每日必出菜品（不受推荐分影响）        |
| 算法参数  | 推荐算法各项加成系数可调                 |


### 六、成本核算

- 基于菜品 BOM（物料清单）自动计算菜品成本
- 食材价格变动时实时重算受影响菜品
- 成本数据驱动毛利率计算，影响推荐排序

### 七、操作日志

所有关键操作（菜品增改、食材调价、菜单发布、用户管理等）自动记录审计日志，包含操作人、时间、变更前后对比。

### 八、任务管理

- 自动从菜单计划生成备料任务（BOM 展开）
- 按角色分配任务，支持状态流转（待处理 → 进行中 → 已完成）
- 权限隔离：不同角色只能操作对应范围的任务
